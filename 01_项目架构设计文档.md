## 二、数据结构定义（models.h）

### 2.1 枚举类型

```c
/* ===== 文化类别 ===== */
typedef enum {
    CAT_TRADITIONAL_CRAFT = 0,  // 传统技艺
    CAT_TRADITIONAL_MUSIC,      // 传统音乐
    CAT_TRADITIONAL_DRAMA,      // 传统戏剧
    CAT_TRADITIONAL_ART,        // 传统美术
    CAT_FOLK_CUSTOM,            // 民俗
    CAT_TRADITIONAL_MEDICINE,   // 传统医药
    CAT_COUNT                   // 类别总数（用于遍历）
} CultureCategory;

/* ===== 保护级别 ===== */
typedef enum {
    PROT_WORLD = 0,             // 世界级
    PROT_NATIONAL,              // 国家级
    PROT_PROVINCIAL,            // 省级
    PROT_CITY,                  // 市级
    PROT_ENDANGERED,            // 濒危
    PROT_COUNT
} ProtectionLevel;

/* ===== 数字化状态 ===== */
typedef enum {
    DIGI_DONE = 0,              // 已数字化
    DIGI_IN_PROGRESS,           // 数字化中
    DIGI_NOT_STARTED,           // 未数字化
    DIGI_COUNT
} DigitalStatus;

/* ===== 用户身份 ===== */
typedef enum {
    ROLE_ADMIN = 0,             // 管理员
    ROLE_USER                   // 普通用户（游客/受众）
} UserRole;

/* ===== 用户状态 ===== */
typedef enum {
    USTAT_NORMAL = 0,           // 正常
    USTAT_BANNED,               // 封禁
    USTAT_CANCELLED             // 注销（等同删除，软删除）
} UserStatus;

/* ===== 档案状态 ===== */
typedef enum {
    ASTAT_NORMAL = 0,           // 正常展示
    ASTAT_HIDDEN,               // 隐藏（下架，不对外展示）
    ASTAT_REMOVED               // 移除（软删除，仅后台保留）
} ArchiveStatus;
```

### 2.2 结构体定义

```c
#define MAX_NAME_LEN     64
#define MAX_ORIGIN_LEN   64
#define MAX_PATH_LEN    128
#define MAX_ACCOUNT_LEN  32
#define MAX_PASSWORD_LEN 32
#define MAX_TIME_LEN     20   // "2026-04-09 14:30:05"

/* ===== 文化档案 ===== */
typedef struct Archive {
    int                id;                          // 项目编号，从10001递增
    char               name[MAX_NAME_LEN];          // 项目名称
    CultureCategory    category;                    // 文化类别
    char               origin[MAX_ORIGIN_LEN];      // 朝代/发源地
    char               inheritor[MAX_NAME_LEN];     // 传承人
    ProtectionLevel    protection;                  // 保护级别
    DigitalStatus      digitalStatus;               // 数字化状态
    int                heatIndex;                   // 国际热度指数 (0-100)
    char               imagePath[MAX_PATH_LEN];     // 封面图片文件名
    int                favoriteCount;               // 收藏量（计算项）
    ArchiveStatus      status;                      // 档案状态：正常/隐藏/移除
    struct Archive    *next;                        // 链表指针
} Archive;

/* ===== 用户 ===== */
typedef struct User {
    char               displayName[MAX_NAME_LEN];   // 用户名（显示用）
    char               account[MAX_ACCOUNT_LEN];    // 账号（登录用，唯一）
    char               password[MAX_PASSWORD_LEN];  // 密码
    UserRole           role;                        // 身份
    UserStatus         status;                      // 用户状态：正常/封禁/注销
    struct User       *next;
} User;

/* ===== 收藏记录 ===== */
typedef struct Favorite {
    char               account[MAX_ACCOUNT_LEN];    // 收藏者账号
    int                archiveId;                   // 收藏的项目编号
    char               time[MAX_TIME_LEN];          // 收藏时间
    struct Favorite   *next;
} Favorite;

/* ===== 全局应用上下文 ===== */
typedef struct AppContext {
    Archive  *archiveHead;     // 档案链表头指针
    User     *userHead;        // 用户链表头指针
    Favorite *favoriteHead;    // 收藏链表头指针
    User     *currentUser;     // 当前登录用户（未登录时为NULL）
} AppContext;
```

### 2.3 计算项说明

| 字段 | 计算方式 | 触发时机 |
|------|---------|---------|
| `favoriteCount` | 遍历收藏链表，统计该项目被收藏次数 | 每次添加/移除收藏后重新计算 |
| 综合传播热度 | `heatIndex × favoriteCount` | 不存储，查询/排序时实时计算 |

> **综合传播热度不存储在结构体中**，因为它随收藏量变化，实时计算更准确且避免数据不一致。

## 四、核心接口设计（函数签名）

### 4.1 链表操作（linkedlist.h）—— 纯数据结构层，不含业务逻辑

```c
/* ===== 档案链表 ===== */
Archive*  archive_node_create(void);                             // 创建空节点（malloc+memset）
void      archive_node_append(Archive **head, Archive *node);    // 尾插一个节点
Archive*  archive_node_find_by_id(Archive *head, int id);        // 按ID查找节点（不过滤状态）
int       archive_node_count(Archive *head);                     // 统计所有节点数（含已移除的）
int       archive_node_max_id(Archive *head);                    // 获取链表中最大ID
void      archive_list_free(Archive *head);                      // 释放整个链表

/* ===== 用户链表 ===== */
User*     user_node_create(void);
void      user_node_append(User **head, User *node);
User*     user_node_find_by_account(User *head, const char *account);  // 按账号查找（不过滤状态）
void      user_list_free(User *head);

/* ===== 收藏链表 ===== */
Favorite* fav_node_create(void);
void      fav_node_append(Favorite **head, Favorite *node);
int       fav_node_remove(Favorite **head, const char *account, int archiveId);  // 真删除（移除节点并free）
int       fav_count_by_archive(Favorite *head, int archiveId);   // 统计某档案的收藏数
int       fav_count_by_account(Favorite *head, const char *account);  // 统计某用户的收藏数
void      fav_list_free(Favorite *head);
```

> **职责边界**：linkedlist.h 只做节点的创建、插入、查找、删除、释放。不做输入校验、不写日志、不保存文件。

### 4.8 内存所有权约定

| 返回类型 | 所有权规则 | 调用者操作 |
|---------|-----------|-----------|
| `Archive*` 单个节点 | 指向链表中的节点 | **不得free** |
| `Archive**` 指针数组 | 数组由malloc分配，元素指向原链表 | **只free数组本身**，不free元素 |
| `char*` 字符串 | 由malloc分配的全新字符串 | **需要free** |
| `AppContext*` | 由app_init分配 | 通过app_shutdown释放 |

## 七、关键设计决策记录

| 决策点 | 结论 | 理由 |
|--------|------|------|
| 综合传播热度存不存 | **不存**，实时算 | 避免收藏变动后数据不一致 |
| 删除方式 | **状态枚举**替代isDeleted | 档案：正常/隐藏/移除；用户：正常/封禁/注销。更灵活 |
| 编号生成 | 读取链表最大ID+1 | 即使有移除状态的记录，编号也不复用 |
| 文件格式 | 文本 `|` 分隔 | 便于调试查看，输入校验禁止文本字段含 `|` |
| 密码存储 | 明文存储 | 课程项目不要求加密，简化实现 |
| 图片管理 | 只存文件名，不嵌入数据库 | 图片随程序目录分发即可 |
| 搜索方式 | `strstr` 模糊匹配 | 任务书要求，输入关键字即可匹配 |
| 数据保存时机 | **每次增删改后立即保存**对应文件 | 防止崩溃丢失数据，不依赖退出时统一保存 |
| 全局数据管理 | **AppContext 统一持有**三条链表 + 当前用户 | 所有函数传 `ctx` 即可，避免散落的全局变量 |
| 排序返回值 | 返回**指针数组**，元素指向原链表 | 调用者只需 `free(array)` 不需释放元素，明确内存归属 |
| linkedlist vs service | linkedlist 只做纯链表操作，service 做业务逻辑 | 职责单一，链表层不写日志不做校验不保存文件 |
